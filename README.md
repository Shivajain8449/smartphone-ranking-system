A Python-powered decision support system that ranks smartphones using the <b>TOPSIS (Technique for Order Preference by Similarity to Ideal Solution)</b> algorithm based on multiple hardware specifications and price.
</p>

---

## ✨ Features

* 📊 Multi-Criteria Decision Making using TOPSIS
* 📱 Rank smartphones based on battery, camera, storage, processor, RAM, and price
* 🖥️ Interactive Streamlit web app with live weight sliders (no config file needed)
* 📂 Upload your own CSV dataset, or use the built-in 10-phone sample
* 🖲️ Standalone CLI script for batch runs, CSV export, and chart generation
* 📈 Automatic ranking visualization
* 🧪 Unit tests with pytest
* 🎯 Beginner-friendly codebase for open-source contributors


## ▶️ Usage

There are two ways to run this project:

### 1. Streamlit Web App (interactive)

```bash
streamlit run app.py
```

This opens a browser-based dashboard where you can:
- Adjust feature weights live using sidebar sliders (must sum to 1.00)
- Choose between the built-in 10-phone dataset or upload your own CSV
- View ranked results, metric cards, and charts instantly

### 2. CLI Script (batch mode)

```bash
python smartphone_ranker.py
```

This runs the TOPSIS pipeline end-to-end with a fixed default weight set, prints a formatted ranking table to the terminal, saves results to `output/final_smartphone_rankings.csv`, and generates a ranking chart.

---

## 📊 Dataset Format

Your CSV file should contain the following columns:

| Column         |
| -------------- |
| SMARTPHONENAME |
| BATTERY        |
| CAMERA         |
| STORAGE        |
| PROCESSOR      |
| RAM            |
| PRICE          |

Example:

| SMARTPHONENAME | BATTERY | CAMERA | STORAGE | PROCESSOR | RAM | PRICE |
| -------------- | ------- | ------ | ------- | --------- | --- | ----- |
| Galaxy X1      | 5000    | 64     | 128     | 2.4       | 8   | 20000 |

## 🧠 How TOPSIS Works

1. Load dataset
2. Normalize data (vector normalization)
3. Apply feature weights
4. Determine ideal best and ideal worst solutions
5. Calculate Euclidean distances from each solution
6. Compute the TOPSIS score
7. Rank smartphones by score

---

## ⚙️ Configuration

Feature weights are configured **live in the Streamlit sidebar** using sliders (Battery, Camera, Storage, Processor, RAM, Price) — they must sum to 1.00 before ranking runs. For the CLI script, default weights are set directly inside `smartphone_ranker.py` and can be edited there.

---

## 📈 Output

The project generates:
- A ranked smartphone list with TOPSIS scores
- A ranking visualization chart
- A CSV export (`output/final_smartphone_rankings.csv`) with columns: `SMARTPHONENAME, BATTERY, CAMERA, STORAGE, PROCESSOR, RAM, PRICE, TOPSIS SCORE, RANK`
